Migrating Services
The steps required in this phase of a Sidecar migration can vary
    depending on the components deployed in the source cluster and planned for
    the destination cluster. In some cases, the order in which you complete
    these migrations should be considered due to dependencies among some
    components. 
Atlas Sidecar migration from HDP to CDPYou can manually migrate Atlas from HDP to CDP. Ranger Sidecar migration from HDP to CDPYou can migrate Ranger policies from HDP to CDP. HDFS Sidecar Migration to CDPYou can migrate data stored in HDFS to CDP.  YARN Capacity Scheduler Sidecar migration from HDP to CDPTo migrate YARN from a HDP cluster you must migrate the Capacity     Scheduler to CDP. The migration includes converting Capacity Scheduler     placement rules to the new JSON format.HBase Sidecar migration from HDP to CDPYou can use HBase replication and snapshots to migrate HBase tables     to the destination cluster. Solr Sidecar migration from HDP to CDPTo migrate Solr from HDP to CDP, you copy the data to the     destination cluster and then re-index the data. Oozie Sidecar migration to CDPTo migrate Oozie to CDP, you must manually migrate workloads and configurations to the     destination cluster. Accumulo Sidecar migration to OpDBYou can migrate the Accumulo service to Operational Database     powered by Apache Accumulo (OpDB) on CDP
