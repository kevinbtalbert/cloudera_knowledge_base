Enable Single User Mode
Configure all clusters to run in single user mode where the Cloudera Manager agent and all service processes run as the same system
user. Only supported for CDH 5.2 and higher. By default, service processes run as distinct users on the system. For example, HDFS DataNodes run as user "hdfs" and HBase RegionServers run as user
"hbase." Enabling "single user mode" configures Cloudera Manager to run service processes as a single user, by default "cloudera-scm", thereby prioritizing isolation between managed services and the
rest of the system over isolation between the managed services.The main benefit of single user mode is that the Agent does not run as root. However, this configuration
complicates installation because directories, which in the regular authentication mode are created automatically by the Agent, must be created manually on every host with appropriate permissions, and
sudo (or equivalent) access must be set up for the configured user.In addition, single user mode may actually be less secure because all processes now run as a single
user.Switching back and forth between single user mode and regular mode is not supported. This feature is deprecated. Contact Cloudera support before enabling this
feature or for assistance with migrating your environment away from this feature.

false
single_user_enabled
true

HOST::CLUSTER::fd_open::TOTAL.

In the above filters the source and target types are entity type strings used within the metric system. ROLE, SERVICE and ALL are wildcards that match all role types, service types and all types
respectively. Metrics are referred to by their user facing names, so counter-based metrics will be in _rate form. The two types of aggregate metrics are TOTAL and
STATISTICAL.After making changes to this field, both the Cloudera Manager Server and the Service and Host Monitors should be restarted.For advanced use only. You could break Cloudera Manager charting
